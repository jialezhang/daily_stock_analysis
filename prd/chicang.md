# 组合仪表盘重构需求文档

## 概述
本文档描述了组合仪表盘的重构需求，目标是通过将界面分为两个独立部分：**目标达成**（左侧）和**持仓概览**（右侧），以改善布局和用户体验。

### 当前布局
当前布局包括：
- 一个显示资产分布明细的表格（如美股、港股、现金等）。
- 一个展示资产类别（权益、现金、加密货币）分布的饼图。
- 一个总结组合总市值、目标市值以及目标完成度的部分。

### 新布局
新布局将分为两个独立的部分：
1. **目标达成（左侧面板）**：
   - 显示以下信息：
     - 初始仓位
     - 收益额（CNY）
     - 收益率
     - 距离目标差值（CNY）
   - 可修改的字段：
     - 初始仓位
     - 目标收益率
     - 计算币种（RMB、USD、HKD）
2. **持仓概览（右侧面板）**：
   - 显示以下信息：
     - 一个饼图，显示资产类别的占比（例如：美股、港股、A股、人民币现金等）。
     - 一个图例，包含以下列：
       - 二级类目
       - 总价值（CNY）
       - 占比总资产
     - 右上角有一个修改按钮，用于打开持仓修改的弹窗。

## 功能需求

### 目标达成（左侧面板）
- **静态展示信息**：
  - **初始仓位**：从 `Config` 中的 `portfolio_initial_capital` 获取。
  - **收益额**：计算公式为 `portfolio.total_value_cny - portfolio.initial_capital`。
  - **收益率**：计算公式为 `(收益额) / portfolio.initial_capital * 100%`。
  - **目标差值**：计算公式为 `portfolio.target_value - portfolio.total_value_cny`。
- **可编辑字段**：
  - **初始仓位**：
    - 默认值：`Config` 中的 `portfolio_initial_capital`。
    - 可通过数字输入框编辑。
    - 修改后更新 `portfolio.initial_capital`，并重新计算相关字段。
  - **目标收益率**：
    - 默认值：`Config` 中的 `portfolio_target_return`。
    - 可通过数字输入框（百分比）编辑。
    - 修改后更新 `portfolio.target_value`，计算公式为 `portfolio.initial_capital * (1 + portfolio_target_return)`。
  - **计算币种**：
    - 默认值：`Config` 中的 `portfolio.output_currency`。
    - 可通过下拉框选择，选项包括：RMB、USD、HKD。
    - 修改后根据汇率（`usd_cny`、`hkd_cny`）更新显示的货币值。

### 持仓概览（右侧面板）
- **饼图**：
  - 显示资产类别（如美股、港股、A股、人民币现金等）的占比。
  - 数据来源：
    - 标签：从 `Portfolio.holdings` 的二级类目获取。
    - 数值：按比例计算 `holding.value_cny / portfolio.total_value_cny`。
- **图例表格**：
  - 列：
    - **二级类目**：`holding.category`
    - **总价值**：`holding.value_cny`
    - **占比总资产**：`holding.value_cny / portfolio.total_value_cny * 100%`
- **修改按钮**：
  - 点击后弹出持仓修改的弹窗。
  - 编辑逻辑保持不变。

## 实现步骤

### 后端改动
1. 修改 `models.py` 中的 `Portfolio` dataclass：
   - 新增字段：`output_currency`（默认值为 "CNY"）。
2. 修改 `runner.py` 中的 `run_portfolio_review` 方法：
   - 支持基于 `output_currency` 的货币转换。
   - 使用 `Config` 中的汇率字段（`usd_cny`、`hkd_cny`）进行转换。
3. 修改 `renderer.py`：
   - 修改报告生成逻辑以适配新布局。
   - 确保所有数值根据选择的货币正确显示。

### 前端改动
1. 更新仪表盘布局：
   - 将当前布局分为两个面板：目标达成（左侧）和持仓概览（右侧）。
   - 在持仓概览部分使用饼图展示资产类别占比。
2. 在目标达成面板中添加可编辑字段：
   - 初始仓位和目标收益率使用数字输入框。
   - 计算币种使用下拉框。
   - 确保编辑这些字段后，更新后端数据并重新计算相关字段。
3. 在持仓概览部分添加修改按钮：
   - 点击后弹出持仓修改弹窗。
   - 复用现有的编辑逻辑。

### 测试
1. **后端测试**：
   - 测试货币转换逻辑。
   - 测试收益额、收益率、目标差值的重新计算。
2. **前端测试**：
   - 验证布局是否符合设计草图。
   - 测试可编辑字段和弹窗的交互行为。
   - 确保编辑字段后数据正确更新。
